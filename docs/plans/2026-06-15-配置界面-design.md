## 1. 背景与目标

当前"配置" Tab 仅为占位文本（[MainWindow.xaml:156-162](../../src/WeflowAgent.App/MainWindow.xaml#L156-L162)）。
后端凭据存储（DPAPI `CredentialStore`）已 TDD 就绪，但前端无录入入口，状态停留在黄色"需配置"无法转绿，闭环存在断点。

本设计实现 FR-CFG-01 要求的"图形化配置界面**覆盖 §6 全部配置项**"，含校验（FR-CFG-02）与凭据掩码录入（FR-CFG-03），打通"录入 → 校验 → 持久化 → 状态刷新"闭环。

## 2. 范围

**本次目标（In Scope）**

- §6 全部约 40 个配置项的图形化录入。
- 非敏感配置模型 `AgentConfig` + `config.json` 持久化（带 `schemaVersion`）。
- 敏感三项（WeFlow Token / 下游 site key / AES 密钥）经 `PasswordBox` 掩码录入，复用既有 DPAPI `CredentialStore`。
- 保存时统一校验（URL / 必填 / 数值 / AES 密钥长度）。
- 保存后刷新主界面黄/绿状态。

**非目标（Out of Scope，预留或后置）**

- FR-CFG-04 "链接变更触发自动重连/热加载" — 依赖尚未实现的 SSE 连接模块，本次**仅预留接口、不真正联动**。
- 连接测试 ping（FR-TEST-03）— 同上，预留。
- FR-SYNC-06 首启向导、FR-CFG-05 导入/导出、FR-SYNC-08 重置、FR-CFG-06 变更记录 — 经确认本次不做。

## 3. 关键设计决策（brainstorm 结论）

| # | 决策 | 取舍理由 |
|---|------|----------|
| D1 | 配置模型用**分组嵌套类**（`AgentConfig` 含 12 个 `*Options` 子类） | 与左侧导航分组、§6 表格一一对应；类型安全；后续各功能模块只取自己那块 |
| D2 | 持久化**双路**：非敏感→`config.json`（明文），敏感三项→`secrets.dat`（DPAPI） | 沿用项目既有约定；敏感不明文落盘（FR-SEC-01） |
| D3 | 界面布局：**左侧分组导航 + 右侧面板** | 十余分组下最清晰、可扩展 |
| D4 | 保存交互：**统一底部"保存"按钮 + 保存时校验** | 事务性清晰；校验集中、不合法定位到字段 |
| D5 | 绑定方式：**WPF 原生数据绑定 + 轻量 ViewModel（`INotifyPropertyChanged`），不引入第三方 MVVM 框架** | 低依赖、贴近现有 code-behind 风格；`PasswordBox` 三项单独 code-behind 读写（安全惯例） |
| D6 | 校验逻辑置于 **Core**（`ConfigValidator`，可 TDD） | 与现有 Core 全程 TDD 一致；界面层不放业务规则 |

## 4. 配置数据模型

### 4.1 结构

```
AgentConfig                 // 根，含 SchemaVersion (int)
├── WeFlow      : WeFlowOptions
├── Downstream  : DownstreamOptions
├── Media       : MediaOptions
├── Catchup     : CatchupOptions
├── Dedup       : DedupOptions
├── Filter      : FilterOptions
├── Heartbeat   : HeartbeatOptions
├── Runtime     : RuntimeOptions
├── Sync        : SyncOptions
├── Install     : InstallOptions
├── Logging     : LoggingOptions
└── Advanced    : AdvancedOptions
```

敏感三项**不在** `AgentConfig` 内，留在既有 `CredentialSet`（`WeflowAccessToken` / `DownstreamSiteKey` / `DownstreamAesKey`）。

## 5. 持久化

| 数据 | 文件 | 路径 | 机制 |
|------|------|------|------|
| `AgentConfig`（非敏感） | `config.json` | [AppPaths.ConfigFile](../../src/WeflowAgent.Core/AppPaths.cs#L49) | `System.Text.Json` 明文 |
| `CredentialSet`（敏感三项） | `secrets.dat` | [AppPaths.SecretsFile](../../src/WeflowAgent.Core/AppPaths.cs#L52) | DPAPI（既有 `CredentialStore`） |

新增 `ConfigStore`（对照 `CredentialStore`）：

- `Load()`：文件不存在 → 返回带默认值的 `AgentConfig`；存在 → 反序列化，并按 `SchemaVersion` 走平滑迁移（复用 [SchemaMigrator](../../src/WeflowAgent.Core/Persistence/SchemaMigrator.cs) 思路，对应 FR-SYNC-09，迁移失败抛异常不静默覆盖）。
- `Save(AgentConfig)`：序列化落盘。

## 6. 校验规则（FR-CFG-02）

`ConfigValidator.Validate(AgentConfig, CredentialSet)` → `ValidationResult { bool IsValid; IReadOnlyList<FieldError> Errors }`。

| 类型 | 规则 |
|------|------|
| URL | `Downstream.BaseUrl` 必须 `https://`（FR-SEC-02）；`WeFlow.HostPort` 为合法 `host:port` |
| 必填 | `BaseUrl`、`SiteKey`、`AesKey` 非空（联调前置三件套） |
| 数值范围 | 各超时/间隔 >0；`RetryCount` ≥0；`MaxLookbackHours`/`MaxItemsPerRun` >0 |
| AES 密钥长度 | `AesKey` 原始串 **≥16 字节 ASCII**（FR-AUTH-02 取前 16 字节，不足则 token 算错） |
| 其它 | `Media.Placeholders` 非空（占位符判定模式下） |

保存时统一校验；失败则左侧导航对应分组标红 + 字段旁提示。

## 7. 界面设计

```
┌─ 配置 Tab ──────────────────────────────────────┐
│ ┌ 导航 ┐  ┌ 右侧面板（随选中分组切换）────────┐ │
│ │WeFlow│  │  Base URL  [____________]          │ │
│ │下游 ●│  │  site key  [••••••••]  (PasswordBox)│ │
│ │媒体  │  │  AES 密钥  [••••••••]  (PasswordBox)│ │
│ │补偿  │  │  请求超时  [15      ] s            │ │
│ │ ...  │  │  ...                               │ │
│ └──────┘  └────────────────────────────────────┘ │
│                              [保存]   (校验失败时分组标红) │
└──────────────────────────────────────────────────┘
```

- 左侧 `ListBox`/导航列出 12 分组；右侧用 `ContentControl` + `DataTemplate`（或分组 UserControl）切换。
- 普通字段双向绑定到 ViewModel；敏感三项 `PasswordBox` 由 code-behind 读写。
- 底部统一"保存"按钮。

## 8. 保存流程

```
点"保存"
  → 收集界面值 → AgentConfig + CredentialSet
  → ConfigValidator.Validate(...)
     ├─ 失败 → 分组标红 + 字段提示，终止
     └─ 通过 → ConfigStore.Save(config)
               CredentialStore.Save(credentials)
               → 重算 StartupStatus（凭据齐 → 绿）
               → 刷新主界面黄/绿
```

## 9. 测试策略（TDD）

Core 可测，先写测试后实现；App 层靠构建 0 警告 + 手动界面验证（参照 verification-before-completion）。

- `AgentConfig`：默认值断言（取自 §6 默认列）。
- `ConfigStore`：缺文件→默认值；`Save→Load` 往返一致；`SchemaVersion` 迁移；迁移失败抛异常。
- `ConfigValidator`：每条规则的通过/失败用例，重点 **AES 密钥 <16 字节报错**、**BaseUrl 非 https 报错**、必填缺失报错。

## 10. 落地拆解（增量，各步独立可交付 + 带测试）

1. `AgentConfig` + 12 个 `*Options` 子类 + 默认值 → 测默认值。
2. `ConfigStore`（Load/Save/默认/迁移）→ 测往返与迁移。
3. `ConfigValidator` → 测各校验规则。
4. App：配置 Tab 改造为 左侧导航 + 右侧分组面板 + 双向绑定。
5. App：保存流程接线（校验 → 双路存储 → 刷新黄/绿）+ 凭据掩码录入。
